Defining Portfolio Archetypes 
A portfolio is not a neutral compilation. It follows an underlying structural archetype whether 
consciously designed or not. 
Defining your archetype before drafting prevents inconsistency, redundancy, and misalignment with 
internship targets. 
An archetype determines: 
● Project order 
● Drawing ratio 
● Technical density 
● Diagram intensity 
● Visual tone 
● Software representation 
Without archetype clarity, a portfolio becomes fragmented. 
Why Archetype Definition is Necessary 
Different firms expect different emphases. 
A portfolio structured for a corporate BIM-heavy firm will not resemble one structured for a boutique 
design studio. 
Before drafting, define: 
● Target firm scale 
● Target design philosophy 
● Target city 
● Target technical depth 
Then choose the archetype accordingly.

Defining Your Archetype 
Before drafting: 
1. Identify target firm category. 
2. Study at least five portfolios in that category. 
3. Study at least three firm projects on ArchDaily. 
4. Check internship postings on LinkedIn. 
5. Write one sentence defining your portfolio identity. 
Example: 
“This portfolio demonstrates balanced conceptual clarity and strong technical discipline suited for 
mid-size institutional firms.” 
That sentence defines your archetype. 
A portfolio archetype is a structural decision. 
Once defined, every page must reinforce it. 
Corporate-Oriented Portfolio Structure 
A corporate-oriented portfolio is engineered for firms that operate at medium to large scale, maintain 
structured workflows, and emphasize documentation precision. These firms expect interns to 
understand drafting standards, coordination systems, and execution logic. 
This structure is not decorative. 
It is disciplined. 
Core Objective 
To demonstrate: 
● Technical reliability 
● Structural awareness 
● Documentation clarity 
● Workflow readiness 
● BIM familiarity 
The portfolio must communicate that the candidate can integrate into a professional system. 
Ideal Project Order 
1. Institutional or large-scale project 
2. Technically resolved academic project 
3. Supporting project with moderate conceptual clarity 
4. Technical sheet section 
5. Skill summary 
The first project must contain: 
● Dimensioned plan 
● Strong section 
● Structural grid 
● Circulation clarity 
Page Structure Model 
Opening Project 
Page 1 
Project introduction (under 80 words) 
Site plan 
Key concept diagram 
Page 2 
Floor plans (dimensioned) 
North arrow 
Scale clearly mentioned 
Page 3 
Longitudinal and cross section 
Bold cut line 
Structural clarity 
Page 4 
Elevation or facade articulation 
Page 5 
Detail drawing (1:20 or similar scale) 
Page 6 
Controlled render (maximum 1–2 views) 
Render must support drawing, not replace it.

Boutique / Design-Led Portfolio Structure 
A boutique-oriented portfolio is structured for design-led studios where architectural identity, spatial 
intelligence, and conceptual clarity carry significant weight. These firms are often smaller in size but 
strong in ideological positioning. 
In such offices, the intern is expected to think, not merely draft. 
The portfolio must therefore communicate design maturity and spatial reasoning without losing 
structural credibility. 
Core Objective 
To demonstrate: 
● Conceptual clarity 
● Sectional intelligence 
● Diagram discipline 
● Material awareness 
● Spatial sensitivity 
The portfolio should feel curated and intentional rather than mechanical. 
Ideal Project Order

2. Section-rich spatial project 
3. Supporting project with material sensitivity

Sustainable / Environmental Portfolio 
Structure 
A sustainable-oriented portfolio is structured for firms that prioritize climate responsiveness, 
environmental logic, material intelligence, and performance-driven design. 
In such offices, design decisions are justified through measurable environmental reasoning rather 
than aesthetic intent alone. 
The portfolio must communicate ecological awareness as structured evidence. 
Core Objective 
To demonstrate: 
● Climate responsiveness 
● Environmental analysis capability 
● Sectional environmental logic 
● Material intelligence 
● Context awareness 
The portfolio should show that sustainability is integrated into design thinking, not appended as an 
afterthought. 
Ideal Project Order 
1. Climate-responsive project 
2. Environmentally analytical project 
3. Supporting project with material sensitivity 
4. Technical detail section 
5. Skill summary (including environmental tools if applicable) 
The first project must clearly demonstrate environmental reasoning.

Cover Page Architecture 
The cover page is the first structural signal of discipline. It does not need visual complexity. It needs 
clarity, hierarchy, and tone alignment with the chosen portfolio archetype. 
A weak cover feels decorative. 
A strong cover feels intentional and controlled. 
The cover page sets expectations for everything that follows. 
Core Objectives of the Cover 
The cover must communicate: 
● Professional identity 
● Portfolio positioning 
● Visual discipline 
● Graphic consistency 
● Tone alignment with target firm type 
It should not: 
● Contain excessive graphics 
● Use unnecessary background textures 
● Overuse typography 
● Include weak project thumbnails 
Minimalism is not emptiness. It is controlled hierarchy. 
Essential Cover Elements 
Include only: 
● Full Name 
● “Architecture Internship Portfolio” (or aligned title) 
● Year 
● Contact Email 
● Optional: Subtle line divider or grid alignment 
Avoid: 
● Long quotes 
● Design philosophy statements 
● Software lists 
● Random renders 
The cover is an introduction, not a content page.

● White or neutral background

● Left-aligned or centered grid

Writing Studio Briefs Professionally 
The studio brief explains context. 
The portfolio must translate it into professional language. 
Most academic briefs are written for juries, not firms. They contain internal terminology, semester 
references, and descriptive language that does not serve professional evaluation. 
The goal is not to copy the studio brief. 
The goal is to extract and refine it. 
Why Rewriting the Brief Matters 
Firms reviewing internship portfolios: 
● Do not know your semester context 
● Do not know your faculty objectives 
● Do not know grading criteria 
They evaluate: 
● Problem clarity 
● Site conditions 
● Constraints 
● Program scale 
● Design response 
The rewritten brief must focus on architectural problem definition. 
Structure of a Professional Brief 
Each project should include a concise contextual summary structured as: 
● Project Type 
● Site Context 
● Core Challenge 
● Design Strategy 
Limit to 60–100 words maximum. 
Avoid narrative storytelling. 
Converting Academic Brief to Professional Format 
Academic Version (Common) 
“This semester project aimed to explore the relationship between space and form while understanding 
climatic responsiveness.” 
Professional Version 
“The project addresses high-density residential living within a coastal urban context. The design 
organizes housing units around a central light court to enhance ventilation while minimizing heat gain.” 
Clarity replaces abstraction. 
Information to Extract from Studio Brief 
Include: 
● Site location 
● Site area 
● Climate type 
● Program size 
● Key design constraint 
● Primary architectural strategy 
Exclude: 
● Faculty commentary 
● Evaluation remarks 
● Personal reflections 
● Semester number 
Keep the content objective.

Line Weight Hierarchy 
Line weight hierarchy is the foundation of drawing clarity. 
It determines what is cut, what is seen beyond, and what is background. 
In internship portfolios, weak line hierarchy immediately signals academic immaturity. Strong line 
hierarchy communicates technical discipline and professional readiness. 
This section defines how to construct and benchmark line weight hierarchy for plans, sections, 
elevations, and details. 
Purpose of Line Weight Hierarchy 
Line weight must communicate: 
● Structural cut 
● Foreground elements 
● Background elements 
● Context 
● Annotation hierarchy 
It must not: 
● Appear uniform 
● Contain random thickness variation 
● Use excessively bold lines 
● Blur structural clarity 
Line hierarchy is architectural logic expressed graphically. 
Primary Hierarchy Structure 
In Plans 
Cut elements (walls, columns, core) 
→ Thickest line 
Visible but not cut elements (doors, windows) 
→ Medium line 
Furniture and secondary ellements 
→ Thin line 
Dimensions and hatches 
→ Very thin line

Cut elements 
→ Bold and dominant

→ Lighter

→ Very light

It silently communicates whether the portfolio is disciplined, experimental, corporate, or conceptual. 
In architectural portfolios, fonts must never dominate content. They must support drawings, structure 
information, and maintain consistency across the entire document. 
The goal is neutrality with precision. 
Why Font Selection Matters 
Fonts influence: 
● Professional perception 
● Readability at print scale 
● Digital clarity 
● Visual hierarchy 
● Portfolio identity 
Weak font selection leads to: 
● Decorative appearance 
● Amateur impression 
● Inconsistent tone 
● Reduced credibility 
Strong font selection is often unnoticed — and that is its success. 
Core Font Selection Principles 
Use: 
● One primary font family 
● Optional secondary weight within same family 
● Maximum two font families 
Avoid: 
● Script fonts 
● Decorative display fonts 
● Handwritten-style fonts 
● More than two type families 
Consistency outweighs uniqueness.

Drawing Annotation and Caption Discipline 
Drawings communicate through hierarchy. 
Annotations must clarify without overwhelming. 
Annotation quality directly influences perceived technical maturity. Weak annotation suggests 
superficial drafting. Strong annotation signals professional discipline. 
This chapter defines standards for labeling, captions, scales, and drawing notes within internship 
portfolios. 
Purpose of Annotation in Internship Portfolios 
Annotations must: 
● Clarify drawing intent 
● Identify spaces and systems 
● Indicate scale and orientation 
● Demonstrate technical awareness 
● Maintain graphic consistency 
They must not: 
● Replace drawings with excessive text 
● Over-explain obvious geometry 
● Compete visually with drawings 
Professional drawings are read quickly. Annotation must assist, not interrupt. 
Levels of Annotation 
Define three levels: 
Primary 
Project title, drawing title 
Secondary 
Room names, major labels 
Tertiary

Avoid more than three levels of text hierarchy inside drawings.

Ethical Representation and Professional 
Integrity 
Academic Honesty, Attribution, and Responsible 
Presentation 
An architectural portfolio is a claim of authorship. 
Every drawing, diagram, model, and render communicates ownership. 
Professional credibility depends not only on design strength but also on ethical clarity. 
This chapter defines how to represent work responsibly and avoid misrepresentation. 
Why Ethical Representation Matters 
Firms evaluate: 
● Technical ability 
● Conceptual clarity 
● Communication discipline 
● Integrity 
Misrepresentation leads to: 
● Loss of trust 
● Immediate rejection 
● Professional reputation damage 
Architecture is collaborative. 
Credit must be transparent. 
Academic Project Attribution 
For studio projects: 
You may present work as: 
“Academic Project – Individual Studio Work” 
If group project: 
Specify: 
“Group Project – Role: Concept Development and Section Drafting” 
Clearly define your contribution. 
Avoid presenting collaborative work as fully individual. 
Internship Work Representation 
If including internship work: 
You must: 
● Confirm permission from firm 
● Remove confidential drawings 
● Avoid publishing sensitive client data 
● Clarify your role 
Example: 
“Internship Contribution – Assisted in working drawing preparation and section detailing.” 
Avoid claiming: 
“Designed the entire project.” 
Transparency builds trust. 
Render Ownership Clarity 
If using: 
● 3D assets 
● Pre-built libraries 
● Online texture packs 
Ensure: 
● You created composition 
● You modeled primary structure 
● You clarify if visualization-only contribution 
Do not claim design authorship for work where you only rendered. 
AI and Tool Usage Transparency 
If using: 
● AI-assisted visualization 
● Parametric tools 
● Automated rendering plugins 
Ensure: 
● Core design logic is yours 
● Diagrams are genuinely constructed 
● Structural reasoning is authentic 
Do not submit AI-generated designs as personal authorship without clarity. 
Professional maturity includes tool transparency. 
Image and Reference Attribution 
If referencing: 
● Site photographs 
● Case study images 
● Precedent diagrams

“Reference Image – ArchDaily”

“Site Image Source: Google Maps”
